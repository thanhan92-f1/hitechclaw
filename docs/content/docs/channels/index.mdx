HiTechClaw supports **9 messaging channels** out of the box. Each channel is implemented as a `ChannelPlugin` that handles incoming messages and sends responses.

## Supported Channels

| Channel | Protocol | Best For |
|---------|----------|----------|
| [Telegram](/docs/channels/telegram) | Bot API (polling) | Teams, personal assistants |
| [Slack](/docs/channels/slack) | Web API + Events | Enterprise workspace |
| [WhatsApp](/docs/channels/whatsapp) | Cloud API (webhook) | Customer support, VN/SEA market |
| [Zalo OA](/docs/channels/zalo) | OA API v3 (webhook) | Vietnamese enterprise |
| [Facebook Messenger](/docs/channels/facebook) | Graph API (webhook) | Customer support, social commerce |
| [MS Teams](/docs/channels/msteams) | Bot Framework | Corporate environments |
| Discord | Bot Gateway | Communities, developer teams |
| [WebChat](/docs/channels/webchat) | Embed widget | Website integration |
| Webhook | Custom HTTP | Third-party systems |

## Architecture

All channels implement the `ChannelPlugin` interface:

```typescript
interface ChannelPlugin {
  id: string;
  platform: ChatPlatform;
  name: string;
  version: string;

  initialize(config: Record<string, unknown>): Promise<void>;
  start(): Promise<void>;
  stop(): Promise<void>;
  send(message: OutgoingMessage): Promise<void>;
  onMessage(handler: (msg: IncomingMessage) => Promise<void>): void;
}
```

## Adding a Channel

1. Go to **Channels** in the sidebar
2. Click **Add Channel** and select a channel type
3. Fill in the required configuration (API tokens, etc.)
4. Click **Test Connection** to verify
5. **Activate** the channel to start receiving messages

Each channel connection is linked to an **Agent Configuration** that determines which AI model and persona processes messages.

## Smart Message Routing

HiTechClaw automatically detects signals in incoming messages and adjusts how they are processed:

### Vision Routing (Image Attachments)

When a user sends an image (Telegram photo, WhatsApp media, etc.), HiTechClaw **hard-switches** to the configured vision model instead of the default text model:

```
ollama-vision → gemini → openai → ollama (fallback)
```

Configure via environment variables:

```bash
VISION_PROVIDER=ollama      # ollama | gemini | openai
VISION_MODEL=llava:13b      # e.g. qwen2.5vl:7b, gemma3:12b
```

> If `VISION_MODEL` is not set, HiTechClaw reuses `LLM_MODEL` (only works if that model supports vision).

### Realtime Search Hint

When a message mentions time-sensitive topics (news, weather, prices, scores), HiTechClaw prepends a routing instruction to the prompt telling the agent to use web-search tools first — no config required.

Keywords detected (Vietnamese + English): `today`, `latest`, `breaking`, `weather`, `price`, `stock`, `hôm nay`, `mới nhất`, `thời tiết`, `tỷ giá`, `kết quả`, `trực tiếp`, and more.

## Webhook Infrastructure

Facebook Messenger, WhatsApp, and Zalo use inbound webhooks. HiTechClaw exposes a unified route:

```
GET  /webhooks/channels/:connectionId   ← Platform verification (hub.challenge)
POST /webhooks/channels/:connectionId   ← Incoming events
```

The `:connectionId` is displayed on the channel card after saving.
