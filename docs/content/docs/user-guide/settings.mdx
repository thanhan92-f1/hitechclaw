# Settings

![Settings](/screenshots/13-settings.png)

The Settings page provides centralized configuration for the entire HiTechClaw platform.

---

## Overview

The Overview section displays real-time system information:

### Status Bar
Four cards showing:
- **Server**: Online/Offline with version (e.g., v2.0.0)
- **LLM Model**: Active model name with available model count
- **Domains**: Loaded domain packs and total skills
- **Uptime**: System uptime with last check timestamp

### Server Information Detail
| Field | Description |
|-------|-------------|
| **Status** | Health check result (ok / error) |
| **Version** | HiTechClaw version |
| **Uptime** | Formatted uptime |
| **Last Check** | Most recent health check timestamp |
| **Platform** | Platform identifier (e.g., "HiTechClaw v2") |
| **Runtime** | Server runtime (Node.js + Hono) |

### Active LLM
| Field | Description |
|-------|-------------|
| **Model** | Currently active model name |
| **Available Models** | Total models accessible |
| **Total Size** | Combined model size on disk |

### Domain Packs
| Field | Description |
|-------|-------------|
| **Loaded Domains** | Number of active domain packs |
| **Total Skills** | Sum of skills across all domains |
| **Status** | Active / Inactive |

## Users & RBAC

Manage users and role-based access control.

### User Management
- View all users with their roles and status
- Create new user accounts
- Edit user details and reset passwords
- Deactivate or delete user accounts

### Roles
HiTechClaw includes pre-defined roles:

| Role | Description |
|------|-------------|
| **Owner** | Full platform access, can manage tenants |
| **Admin** | Administrative access, can manage users and settings |
| **Manager** | Can manage agents, workflows, and knowledge base |
| **Operator** | Can use chat and view analytics |
| **Viewer** | Read-only access to conversations and analytics |

### Permissions
60 granular permissions organized into groups:
- **Chat**: Send messages, view history, manage sessions
- **Knowledge**: Upload, edit, delete documents
- **Agents**: Create, edit, delete, assign agents
- **Workflows**: Create, edit, run, delete workflows
- **Analytics**: View dashboards, export data
- **Settings**: Modify platform configuration
- **Admin**: Manage users, roles, tenants, API keys
- **Channels**: Connect, configure, disconnect channels
- **Marketplace**: Install, uninstall, manage skills
- **Security**: View audit logs, manage retention

### Custom Roles
Create custom roles by selecting specific permissions:
1. Click **+ New Role**
2. Name the role
3. Select permissions from each group
4. Save — the role is immediately available for assignment

## LLM / Models

Configure AI model providers and manage available models.

### Supported Providers

| Provider | Models |
|----------|--------|
| **Ollama** | Any locally hosted model (qwen, llama, codellama, mistral, etc.) |
| **OpenAI** | GPT-4o, GPT-4o Mini, GPT-4 Turbo |
| **Anthropic** | Claude Sonnet 4, Claude 3.5 Sonnet, Claude 3 Haiku |
| **DeepSeek** | DeepSeek Coder, DeepSeek Chat |
| **xAI** | Grok-2 |
| **OpenRouter** | Any model available on OpenRouter (100+) |
| **Perplexity** | pplx-7b, pplx-70b, llama-sonar series |
| **Google** | Gemini 2.0 Flash, Gemini 1.5 Pro |
| **Groq** | LLaMA 3.3-70b, Mixtral (ultra-fast inference) |
| **Mistral** | Mistral Large, Mistral Medium |
| **HuggingFace** | Any hosted inference model via HuggingFace API |
| **Custom** | Any OpenAI-compatible endpoint with custom base URL |

### Tenant Model Defaults

The tenant-wide default provider and model are stored in the database and exposed via `GET /api/settings`. Agents that do not specify their own provider/model automatically inherit these defaults.

The **Agent Builder** reads `GET /api/settings → modelDefaults` on load and pre-fills the Provider and Model fields accordingly. The tenant's provider is marked **★ Tenant Default** in the provider dropdown.

```json
// GET /api/settings response (excerpt)
{
  "modelDefaults": {
    "provider": "ollama",
    "model": "qwen2.5:14b",
    "temperature": 0.7,
    "maxTokens": 2048
  }
}
```

To change the tenant defaults:
1. Go to **Settings → LLM / Models**
2. Select a **Provider** from the 12 available options
3. Enter or select the **default model**
4. Set **Temperature** and **Max Tokens** as appropriate
5. Save — all agents without per-agent overrides will use the new defaults

### Per-Agent Override

Each agent created in the **Agent Builder** can override the tenant defaults. Choose a different provider or model in the agent's **Settings** tab. A warning is shown if you pick a provider that differs from the tenant default (to remind you a separate API key may be required).

### Configuration
For each provider:
1. Enter the **API Key** (or base URL for Ollama / Custom)
2. Set as the **tenant default** if desired
3. Test the connection
4. Save

### Ollama Integration
If Ollama is running locally, HiTechClaw auto-detects available models. Manage them from the **Ollama Models** page in the sidebar.

## RAG Engine

Configure the document processing and retrieval pipeline.

| Setting | Description | Default |
|---------|-------------|---------|
| **Chunk Size** | Characters per text chunk | 1000 |
| **Chunk Overlap** | Overlapping characters between chunks | 200 |
| **Embedding Model** | Model for vector embeddings | Ollama default |
| **Similarity Threshold** | Minimum cosine similarity for results | 0.7 |
| **Top K** | Maximum results returned per query | 5 |
