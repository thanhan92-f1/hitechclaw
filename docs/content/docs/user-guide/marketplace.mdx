# Skill Marketplace

![Skill Marketplace](/screenshots/08-marketplace.png)

The Marketplace is where you discover and install AI skills. Skills extend agent capabilities with domain-specific tools, prompts, and processing logic.

---

## Browsing Skills

### Category Filters
Use the filter bar to narrow results by category:

| Category | Examples |
|----------|---------|
| **All** | Show all 24 available skills |

| **Healthcare** | Drug interactions, ICD-10 coding, clinical notes |
| **Security** | Threat analysis, vulnerability scanning |
| **AI / ML** | Model training, data analytics |
| **Integrations** | External service connectors |
| **Analytics** | Data analysis, reporting |

### Search
Use the search bar at the top to find skills by name, description, or keyword.

---

## Skill Card Details

Each skill card displays:

| Field | Description |
|-------|-------------|
| **Name** | Skill identifier (e.g., "Drug Interaction Checker") |
| **Author** | Creator (e.g., "xClaw") |
| **Domain** | Industry category (e.g., "Healthcare & Medical") |
| **Version** | Current version (e.g., v1.0.0) |
| **Description** | What the skill does |
| **Tags** | Category labels for filtering |
| **Rating** | Community rating (stars) |
| **Downloads** | Installation count |
| **Status** | "Installed" badge if already active |

---

## Installing a Skill

1. Find the skill you want in the marketplace
2. Click the skill card to view details
3. Click **Install** to add it to your platform
4. The skill becomes available for agent assignment

Installed skills show a green **Installed** badge on their card.

## Assigning Skills to Agents

After installing a skill:
1. Go to **Agents** → select an agent → **Edit**
2. Navigate to the **Skills** tab
3. Toggle the installed skills you want this agent to use
4. Save the agent

Each agent can have a different combination of skills, allowing you to create specialized agents for different tasks.

---

## Skill Architecture

Skills are built using the `defineSkill()` helper from `@xclaw-ai/core`:
- **Tools**: Function definitions the LLM can call
- **Prompts**: Domain-specific system prompt extensions
- **Knowledge**: Pre-loaded data (e.g., drug interaction databases)
- **Validators**: Input/output validation logic

Skills are loaded dynamically at runtime and can be hot-swapped without restarting the server.
