### 2. Train Model

Train a machine learning model with your own data.

#### Step-by-Step
1. Switch to the **Train Model** tab
2. **Select Algorithm** — Choose from the 12 available algorithms
3. **Upload Data** — Upload a CSV file with your training data
4. **Configure Features**:
   - Select **input columns** (features)
   - Select **target column** (label/output)
   - Set **train/test split** ratio (default: 80/20)
5. **Set Hyperparameters** — Algorithm-specific settings
6. Click **Train** — Monitor training progress
7. View **Results**:
   - Accuracy / R² score
   - Confusion matrix (classification)
   - Feature importance
   - Validation metrics

---

### 3. AutoML

Automatic model selection and hyperparameter tuning.

#### Step-by-Step
1. Switch to the **AutoML** tab
2. Upload your dataset (CSV)
3. Select the **target column**
4. Choose the **task type** (classification or regression)
5. Click **Run AutoML**
6. The system automatically:
   - Tries multiple algorithms
   - Tunes hyperparameters
   - Cross-validates results
   - Ranks models by performance
7. Review the **leaderboard** — best model appears on top
8. Click **Deploy** on the best model to save it to the registry

## Reinforcement Learning — Bandit Skill Selection

HiTechClaw includes a **Multi-Armed Bandit** system that uses reinforcement learning to automatically select the best skill/tool for each user query. Instead of static rule-based selection, the agent learns over time which skills produce better results.

### How It Works

Each skill is treated as an "arm" in a multi-armed bandit problem. When a skill is used, the system collects a **reward signal** (from user feedback 👍/👎 or tool execution success/failure). Over time, the bandit algorithm learns to prefer skills with higher rewards while still exploring less-used skills.

### Strategies

| Strategy | Description | Best For |
|----------|-------------|----------|
| **UCB1** (default) | Upper Confidence Bound — balances exploitation of known-good skills with exploration of uncertain ones | General use, good convergence |
| **Epsilon-Greedy** | Picks the best-known skill with probability `1-ε`, random otherwise. Epsilon decays over time | Simple scenarios, fast bootstrapping |
| **Thompson Sampling** | Bayesian approach — samples from Beta distribution per skill to decide | Best theoretical performance, handles uncertainty well |

### Reward Signals

| Source | Signal | Reward Value |
|--------|--------|--------------|
| **User feedback** | 👍 thumbs up | `1.0` |
| **User feedback** | 👎 thumbs down | `0.0` |
| **Tool execution** | Success | `1.0` |
| **Tool execution** | Failure/error | `0.0` |
| **Composite** | Weighted combination of above | `0.0–1.0` |

### Usage Example

```typescript
import { SkillManager } from '@hitechclaw-ai/core';
import { BanditSelector } from '@hitechclaw-ai/ml';

// Create a bandit with UCB1 strategy
const bandit = new BanditSelector('tenant-123', { strategy: 'ucb1' });

// Attach to skill manager
const skills = new SkillManager();
skills.setSelector(bandit);

// Skills are auto-registered as bandit arms
skills.register(weatherSkill);
skills.register(searchSkill);

// Agent selects the best skill via RL
const bestSkill = skills.selectBestSkill();

// Get tools ordered by RL ranking (best skills first)
const rankedTools = skills.getRankedTools();

// After tool execution, send reward
skills.rewardSkill('weather', 1.0, 'tool-success');

// After user feedback
skills.rewardSkill('search', 0.0, 'user-feedback');
```

### Bandit State Persistence

The bandit state (arm statistics, epsilon, pull counts) can be exported and saved to MongoDB for cross-session persistence:

```typescript
// Export state
const state = bandit.getState();
await saveBanditState(state); // your persistence logic

// Restore state on startup
const saved = await loadBanditState('tenant-123');
bandit.loadState(saved);
```

### Key Concepts

| Concept | Meaning |
|---------|----------|
| **Arm** | A skill or tool that can be selected |
| **Pull** | One selection + execution of an arm |
| **Reward** | Feedback signal (0 = bad, 1 = perfect) |
| **Exploitation** | Picking the arm with the best known reward |
| **Exploration** | Trying less-used arms to discover better options |
| **Epsilon (ε)** | Probability of random exploration (decays over time) |
| **UCB** | Upper Confidence Bound — adds exploration bonus to uncertain arms |

### Tips

- Start with **UCB1** — it provides the best balance without tuning
- **Thompson Sampling** works best with many skills and high traffic
- **Epsilon-Greedy** is simplest to understand and debug
- The system needs ~50–100 interactions per skill to converge
- Bandit state is per-tenant — each tenant learns independently
- Use `getRanking()` to inspect which skills are performing best

## Tips

- Start with simpler models (Linear/Logistic Regression) before trying ensemble methods
- Ensure your data is clean — remove null values and outliers before training
- Use AutoML for first-pass model selection, then fine-tune manually
- Compare at least 3 algorithms on the same dataset
- Use cross-validation for more reliable accuracy estimates
