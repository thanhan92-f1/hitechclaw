HiTechClaw can be installed from source or run with Docker.

## From Source (Development)

Best for development and customization.

```bash
git clone --recurse-submodules https://github.com/thanhan92-f1/hitechclaw.git
cd hitechclaw
npm install
cp .env.example .env
# Edit .env with your API keys
npm run dev
```

> If you already cloned without `--recurse-submodules`, initialize submodules with:
> ```bash
> git submodule update --init --recursive
> ```

## Docker Compose (Recommended for Production)

The fastest way to deploy HiTechClaw.

```bash
git clone --recurse-submodules https://github.com/thanhan92-f1/hitechclaw.git
cd hitechclaw
cp .env.example .env
# Edit .env with your API keys

docker compose up -d
```

HiTechClaw will be available at `http://localhost:18789`.

### Docker Compose Configuration

```yaml title="docker-compose.yml"
services:
  hitechclaw:
    build: .
    ports:
      - "18789:18789"
    env_file:
      - .env
    volumes:
      - ./data:/app/data
    restart: unless-stopped
```

## Docker Build Only

Build the production Docker image:

```bash
docker build -t hitechclaw .
docker run -p 18789:18789 --env-file .env hitechclaw
```

The multi-stage Dockerfile:
1. **Base** — Installs dependencies
2. **Builder** — Compiles all TypeScript packages
3. **Server** — Minimal production image with compiled output

## npm Packages

Install individual packages for use in your own projects:

```bash
npm install @hitechclaw-ai/core @hitechclaw-ai/shared
```

| Package | Description |
|---------|-------------|
| `@hitechclaw-ai/shared` | Type definitions, constants, interfaces |
| `@hitechclaw-ai/core` | Agent engine, LLM router, memory, tools |
| `@hitechclaw-ai/skills` | Pre-built AI agent skill packs |
| `@hitechclaw-ai/gateway` | HTTP/WebSocket gateway server |
| `@hitechclaw-ai/cli` | Command-line interface |
