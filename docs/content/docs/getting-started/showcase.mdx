## 1. Login

![Login](/screenshots/02-login.png)

Navigate to `http://localhost:3000` to access the login page.

### How to Log In
1. Open `http://localhost:3000` in your browser
2. Enter your **Email** (default: `admin@hitechclaw.io`)
3. Enter your **Password** (default: `password123`)
4. Click **Sign In**

The left panel displays the HiTechClaw logo, version badge (v2.0), and platform highlights:

- **Enterprise Security** — RBAC with 60+ permissions

> The admin account is automatically created during initial seed. Change the password in **Settings → Users & RBAC** after first login.

## 4. Knowledge Base

![Knowledge Base](/screenshots/05-knowledge.png)

Manage documents that power RAG (Retrieval-Augmented Generation).

### How to Upload a Document
1. Navigate to **Knowledge Base** in the sidebar
2. Click **+ Add Document**
3. Choose input method:
   - **Text**: Paste content directly
   - **File Upload**: Upload PDF, TXT, DOCX, MD, CSV, JSON
   - **Web URL**: Fetch and index a web page
4. Select a collection (or use "General")
5. Click **Save** — document is automatically chunked and embedded

### Collections
- **General**: Default collection for uncategorized documents
- Click **+** in the Collections sidebar to create custom collections
- Filter documents by clicking a collection name

### Statistics
| Metric | Description |
|--------|-------------|
| **Documents** | Total uploaded documents |
| **Enabled** | Documents active for RAG retrieval |
| **Chunks** | Text segments after splitting |
| **Collections** | Number of collections |

### Document Management
- **Enable/Disable**: Toggle documents on/off without deleting
- **Delete**: Permanently remove a document and its chunks
- **Analytics**: View retrieval frequency and relevance scores

## 5. RAG Search

![RAG Search](/screenshots/15-search.png)

Test retrieval quality by searching your knowledge base with natural language.

### How RAG Search Works
The interface shows a 4-step pipeline:
1. **Query Embedding** — Your query is converted to a vector embedding
2. **Similarity Search** — Finds most similar document chunks by cosine distance
3. **Context Building** — Top chunks form the context for the LLM
4. **Augmented Answer** — LLM generates an answer grounded in retrieved context

### How to Search
1. Navigate to **RAG Search** in the sidebar
2. Enter a natural language query in the search box
3. Select collection filter (default: **All Collections**)
4. Set **Top K** results count (default: **Top 5**)
5. Click **Search**
6. Results show source document, chunk text, and relevance score

### History
Click **History** (top right) to view previous searches.

## 7. Workflows

![Workflows](/screenshots/07-workflows.png)

Visual workflow builder with drag-and-drop canvas.

### Creating Your First Workflow
1. Click **+ New Workflow** or **Create Your First Workflow**
2. Enter a **Name** and **Description**
3. Select a **Trigger** type:
   - **Manual**: Run on demand
   - **Cron Schedule**: Run on a schedule (e.g., `0 9 * * 1-5`)
   - **Message Event**: Trigger on keyword/domain/channel match
   - **Webhook**: Receive external HTTP requests
4. Drag **nodes** from the palette onto the canvas
5. Connect nodes by dragging output→input ports
6. Configure each node's parameters
7. Click **Run** to test, then **Save** and activate

### Node Types (16)
- **AI**: LLM Call, RAG Query, Sentiment Analysis, Text Classification
- **Logic**: Condition, Switch, Loop, Delay
- **Action**: HTTP Request, Send Message, Send Email, Code Execute
- **Data**: Data Transform, JSON Parse, Template, Aggregate

Three cards at the top:
- **Status**: Running / Offline (with Ollama version)
- **Models**: Total installed model count

A list of all locally installed models showing:

### How to Pull a New Model
1. Navigate to **Ollama Models** in the sidebar
2. Enter a model name in the "Pull New Model" input (e.g., `llama3.2:3b`)
3. Click the download button or select from the suggested models grid:
   - `llama3.2:3b`, `qwen2.5:7b`, `qwen2.5:3b`, `gemma2:2b`, `phi3.5:3.8b`, `mistral:7b`

5. Click **Use** to activate the model

1. Click **Use** on any installed model
2. The model becomes the default for all AI interactions
3. The previous active model remains installed
